User Specific HTML Pages

To integrate other tools into AvNav there is an option to integrate "User Apps" (or addons). If such addons are configured, you will have an app button  on the main page. By clicking this button there will be a new page with one of the configured add ons being displayed.

In the picture you see the signalK web interface - that has been automatically configured by the signalk plugin.

For each of your user apps you need an icon. Those icons should be uploaded to the images folder via Files/Download , subpage images.

Additionally you need an external URL or a HTML page that has been uploaed into the user dir.

Via Settings , User Apps you reach the configuration page for user apps. Here you can create new entries (or edit/delete existing ones). The Icon you select here will become the icon on the button on the user app page.


With the  "+" button you can create a new entry.

If you select "internal" you will be prompted with a list of the available HTML pages in the user directory.

When leaving the title entry empty you will not see a title bar on the user app page if this entry is active.

Remark: When you create a user app (or modify/delete one) the avnav server will modify and write back it's config file (avnav_server.xml). If something goes wrong in this process and the server cannot start up the next time it will fall back to a config file with the extension ".ok" that it creates on every successful start up.

Via the button you can directly go to to the user app page with the selected app being active.

The picture shows the result of the example entry.

If you created a user app configuration from a HTML page in the user directory and you delete this HTML file via AvNav the user app configuration will be deleted, too.

If you delete a used icon the user app will not be visible at the user app page any more - but you can still reach the configuration and e.g. select another icon.
